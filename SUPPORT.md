# Support

Use GitHub issues for public support that can help future contributors or users.

Open an issue for:

- Reproducible bugs.
- Local setup problems.
- Documentation gaps.
- Feature requests.
- Questions about expected behavior.

Do not open a public issue for a vulnerability. Follow [SECURITY.md](./SECURITY.md).

## What To Include

For bugs, include:

- Steps to reproduce.
- Expected behavior.
- Actual behavior.
- Browser, operating system, and device details when relevant.
- Logs, screenshots, request IDs, or preview URLs when useful.

For setup problems, include:

- Node.js and npm versions.
- The command that failed.
- Full error output.
- Whether Wrangler is authenticated.
- Whether the issue involves local D1, remote R2, KV, or Cloudflare Images.

For licensing or commercial-use questions, start with [LICENSE](./LICENSE).
